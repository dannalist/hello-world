## 1. 前提条件（Prerequisites）
列出执行操作前必须满足的条件，避免操作受阻：
- 硬件要求（例：服务器内存≥8GB，硬盘剩余空间≥50GB）
- 软件要求（例：已安装 Java JDK 1.8+，操作系统为 CentOS 7.6）
- 权限要求（例：拥有服务器 root 账号或管理员权限）
- 准备材料（例：XX 软件安装包、配置文件模板、license 密钥）

## 2. 背景信息（Background）
背景信息是任务执行前的前置说明，用于让读者理解 “为何做”“适用场景”“前置条件”，不包含具体操作步骤。
- 明确任务目的，说明执行该任务能达成的结果或解决的问题。
- 界定适用范围，比如适用的用户角色、产品版本、使用场景。
- 列出前置条件，包括执行任务前需完成的准备、必备权限、所需工具 / 资源。

## 3. 操作步骤（Steps）
按 “时间顺序 + 逻辑递进” 拆分步骤，每步仅含一个核心操作：
1. 执行前置准备操作（例：登录服务器，进入 /usr/local/software 目录）
2. 解压安装包（例：运行命令 “tar -zxvf XX-3.0.tar.gz”，等待解压完成）
3. 配置核心参数（例：编辑配置文件 config.ini，设置端口号为 8080、数据库连接地址为 jdbc:mysql://192.168.1.100:3306/xxdb）
4. 启动服务（例：执行脚本 “./start.sh”，观察控制台输出 “Service started successfully”）
5. 验证服务状态（例：访问 http:// 服务器 IP:8080，确认页面正常加载）

## 4. 预期结果（Expected Results）
明确操作成功后的可验证效果，便于执行者确认：
- 服务状态：XX 服务进程正常运行，无报错日志
- 功能验证：可通过 XX 命令（例：“ps -ef | grep XX”）或页面访问确认功能可用
- 额外指标：服务启动时间≤30 秒，占用内存≤2GB

## 5. 常见问题与解决办法（Troubleshooting）
列出操作中高频出现的问题及应对方案：
- 问题 1：解压安装包时提示 “权限不足”
解决办法：切换 root 账号（执行 “su root”），或添加执行权限（“chmod +x XX-3.0.tar.gz”）
- 问题 2：启动服务后页面无法访问
解决办法：检查防火墙是否开放 8080 端口（“firewall-cmd --list-ports”），未开放则执行 “firewall-cmd --permanent --add-port=8080/tcp” 并重启防火墙
