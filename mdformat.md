# Markdown常见格式
核心结论：Markdown 格式涵盖文本修饰、结构排版、媒体嵌入等核心功能，语法简洁通用，适配多数编辑器和平台。
## 一、文本基础修饰
用于突出重点、调整格式，是最常用的语法：
- 标题：用 # 表示，从 # 一级标题 到 ###### 六级标题（# 后需加空格）
- 加粗：**加粗内容** 或 __加粗内容__，渲染为 **加粗内容**
- 斜体：*斜体内容* ，渲染为 *斜体内容*
- 粗斜体：***粗斜体内容***，渲染为 ***粗斜体内容***
- 删除线：~~删除线内容~~，渲染为 ~~删除线内容~~
- 下划线：++下划线内容++（部分编辑器支持），或用 HTML 标签 <u>下划线内容</u>，渲染为 <u>下划线内容</u>
- 代码片段：单行用 `代码内容`，渲染为 `代码内容`；多行用 ``` 包裹（可指定语言）

## 二、列表排版

用于梳理步骤、分类内容，支持有序和无序：

### 1. 无序列表
用 - * 或 + 开头（后加空格），可嵌套
> markdown
> - 一级列表项1
>   - 二级列表项1.1
>   - 二级列表项1.2
> * 一级列表项2
> + 一级列表项3
{.is-info}

### 2. 有序列表
用数字 + . 开头（后加空格），序号可自动递增，支持嵌套
> markdown
> 1. 第一步操作
>    1. 子步骤1.1
>    2. 子步骤1.2
> 2. 第二步操作
> 3. 第三步操作
> {.is-info}
{.is-info}

### 3. 任务列表（部分编辑器支持）
用 - [ ] 表示未完成，- [x] 表示已完成
> markdown
> - [x] 完成文档编写
> - [ ] 检查格式正确性
> - [ ] 提交审核
{.is-info}

## 三、段落与布局
用于调整文本结构、优化阅读体验：
- 换行：行尾加 2 个空格，或直接空一行（推荐后者）
- 分段：空一行即可分隔两个段落
- 引用：用 > 开头，支持嵌套（多组 > 叠加）
> 请问 Markdwon 怎么⽤？ - ⼩⽩
> {.is-info}
{.is-info}

>> ⾃⼰看教程！ - 愤青
{.is-info}

> >> 教程在哪？ - ⼩⽩
{.is-info}

- 分割线：用 --- *** 或 ___（单独一行，前后空行），渲染为 ---
- 缩进：直接用空格（4 个空格 = 1 个缩进），或用 HTML 标签 `<blockquote>` 辅助
  
## 四、链接与媒体
用于插入外部资源、本地文件或多媒体：
  
### 1. 链接
- 普通链接：[链接文本](链接地址 "可选标题")，例：[Markdown 指南](https://www.markdownguide.org "官方指南")
- 锚点链接：[跳转文本](#目标标题)（需目标标题存在，例：[跳转至列表部分](#列表排版)）
- 参考链接：先定义 [链接文本][参考ID]，再在文末标注 [参考ID]: 链接地址 "可选标题"
### 2. 图片
- 语法：![图片alt文本](图片地址 "可选标题")，本地图片可填路径（相对 / 绝对）
- 调整大小（部分编辑器支持）：用 HTML 标签 `<img src="图片地址" width="300" height="200">`
### 3. 视频（部分平台支持）
直接插入链接，或用 HTML 标签 `<video src="视频地址" controls></video>`
## 五、表格
用于展示结构化数据，支持对齐、嵌套（详细语法见前文）：
- 基础语法：用 | 分隔列，- 分隔表头与内容，: 控制对齐（左对齐 : 靠左，居中 : 两侧加，右对齐 : 靠右）  

| 表头1 | 表头2 | 表头3 |
| :--- | :---: | ---: |
| 左对齐 | 居中 | 右对齐 |

## 六、特殊功能（扩展语法）
部分编辑器（如 Typora、VS Code 插件）支持的进阶功能：
- 脚注：文本内容[^脚注ID]，文末标注 [^脚注ID]: 脚注说明
- 表情符号：:emoji名称:（例 :smile: 渲染为 😄，需编辑器支持）
- 数学公式：用 $ 包裹（单行 $公式$，多行 $$公式$$，需加载 LaTeX 支持）
- 图表：通过 Mermaid 语法实现（例流程图、时序图），用 ```mermaid 包裹  
  
## 七、注意事项
- 语法兼容性：部分扩展功能（如脚注、任务列表）并非所有平台支持，需确认编辑器 / 发布平台特性
- 特殊字符转义：遇到 # * [ ] ( ) ~ !等符号需显示原义时，用` 转义（例 \# 不被解析为标题）
